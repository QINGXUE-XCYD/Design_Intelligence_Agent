# 实验4中文结果分析：充电竞争对系统效率的影响

## 1. 实验目的

本实验用于研究在多机器人协同清扫过程中，充电资源是否会成为系统效率瓶颈。  
具体来说，我们重点考察两个因素：

- 机器人数量 `num_agents`
- 充电站容量 `charging_station_capacity`

其中，充电站容量表示同一充电站在同一时刻允许多少个机器人同时充电。

## 2. 实验设置

本实验采用二维对比设计：

- `num_agents in [2, 3, 4]`
- `charging_station_capacity in [1, 2]`

其余主要控制变量如下：

- 协作策略：`shared_map_reservation`
- 传感器模式：`manhattan`
- 传感器范围：`3`
- 地图大小：`30 x 30`
- 障碍密度：`0.15`
- 目标覆盖率：`0.95`
- 动态障碍数量：`0`

为了更明显地触发充电行为，本实验使用了相对更严格的电池参数：

- `battery_capacity = 100.0`
- `low_battery_threshold = 22.0`
- `recharge_rate = 12.0`
- `battery_safety_margin = 15.0`
- `additional_charging_stations = 0`

每组条件运行 `10` 个不同随机种子，并对结果取均值和标准差。

## 3. 主要结果

各组实验的主要均值结果如下：

| 条件 | 成功率 | 平均总步数 | 平均等待充电步数 | 平均充电事件数 |
| --- | --- | ---: | ---: | ---: |
| 2 robots, capacity=1 | 1.00 | 790.7 | 71.7 | 79.1 |
| 2 robots, capacity=2 | 1.00 | 748.5 | 0.0 | 19.5 |
| 3 robots, capacity=1 | 1.00 | 486.7 | 107.7 | 81.0 |
| 3 robots, capacity=2 | 1.00 | 451.0 | 16.1 | 29.8 |
| 4 robots, capacity=1 | 1.00 | 354.1 | 104.3 | 74.6 |
| 4 robots, capacity=2 | 1.00 | 330.9 | 26.3 | 37.2 |

从结果可以看出，本次实验中所有条件的 `success_rate` 都为 `1.0`，说明当前参数设置既能够稳定完成任务，又能较明显地体现充电竞争带来的差异。

## 4. 指标含义说明

### 4.1 `total_charge_wait_steps`

该指标表示所有机器人在到达充电站后，由于当前没有获得充电权限而原地等待的总步数。  
这个值越大，说明充电资源竞争越激烈。

### 4.2 `total_charging_events`

该指标表示系统中机器人进入 `charging` 状态的总次数。  
它不是简单的“返航次数”，而是“进入充电状态”的次数。

在当前实现中，如果机器人在充电站上因为容量限制经历了“等待-充电-再等待-再充电”的切换，那么每次重新进入 `charging` 状态都会增加一次 `charging_event`。因此这个指标也能反映充电过程是否被频繁打断、是否较为碎片化。

### 4.3 `total_steps`

该指标表示系统达到目标覆盖率 `0.95` 时所需的总步数。  
它反映的是整体任务完成效率，值越小表示效率越高。

## 5. 结果分析

### 5.1 充电站容量增大能够显著降低等待时间

在相同机器人数量下，将充电站容量从 `1` 提升到 `2` 后，等待充电步数都明显下降：

- `2 robots`: `71.7 -> 0.0`
- `3 robots`: `107.7 -> 16.1`
- `4 robots`: `104.3 -> 26.3`

这说明当前系统中，充电站容量确实会直接影响机器人之间的资源竞争情况。当容量较小时，机器人虽然能够到达充电站，但并不能立即开始充电，而是需要在原地等待，从而造成额外时间开销。

### 5.2 等待时间减少后，整体任务效率同步提升

在相同机器人数量下，容量从 `1` 提升到 `2` 后，总步数也都有所下降：

- `2 robots`: `790.7 -> 748.5`
- `3 robots`: `486.7 -> 451.0`
- `4 robots`: `354.1 -> 330.9`

这表明充电竞争不仅仅影响局部行为，还会影响整体任务效率。机器人等待充电的时间越多，整个系统完成目标覆盖率所需的总时间也越长。

### 5.3 机器人数量增加时，系统效率提高，但充电竞争也更容易出现

在容量固定时，随着机器人数量从 `2` 增加到 `4`，总步数明显下降，说明多机器人协同本身确实能提升探索与清扫效率。  
但与此同时，更多机器人也意味着更多潜在充电需求，因此在容量不足时，等待充电现象会更明显。

例如在 `capacity = 1` 时：

- `2 robots`: `mean_wait = 71.7`
- `3 robots`: `mean_wait = 107.7`
- `4 robots`: `mean_wait = 104.3`

而在 `capacity = 2` 时：

- `2 robots`: `mean_wait = 0.0`
- `3 robots`: `mean_wait = 16.1`
- `4 robots`: `mean_wait = 26.3`

可见，提高容量后，随着机器人数量增加所带来的额外竞争被明显缓解了。

### 5.4 充电事件数下降说明充电过程更加连续

从结果看，容量为 `1` 时，`charging_events` 普遍较高；容量为 `2` 时则显著降低。例如：

- `3 robots`: `81.0 -> 29.8`
- `4 robots`: `74.6 -> 37.2`

这说明当容量较小时，机器人更容易出现“等待-充电-等待-再充电”的碎片化过程；而当容量增大后，机器人更容易一次性完成连续充电，因此进入 `charging` 状态的次数反而减少。

## 7. 实验结论

本实验表明，在多机器人协同清扫系统中，充电站容量会显著影响系统运行效率。

- 当充电站容量较小时，机器人更容易在充电站处发生等待，导致 `total_charge_wait_steps` 明显增加
- 等待时间增加会进一步拖慢整体任务完成速度，使 `total_steps` 上升
- 将充电站容量从 `1` 提升到 `2` 后，可以明显缓解资源竞争，提高整体效率
- 在机器人数量增加时，系统协作效率虽然提高，但对充电资源的需求也更强，因此合理配置充电站容量十分重要

因此可以认为，充电资源配置是影响多机器人系统实际性能的重要因素之一。适当提高充电站并发容量，有助于降低资源冲突、减少等待时间，并提升整体任务执行效率。
