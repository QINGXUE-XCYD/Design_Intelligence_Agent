# 实验3A结果分析：传感器范围对系统性能的影响

## 1. 实验目的

本实验旨在研究传感器感知范围变化对多机器人清扫系统性能的影响。  
在保持协作策略、机器人数量和环境参数不变的前提下，仅改变传感器范围，观察系统在覆盖效率、重复访问和能耗方面的变化。

## 2. 实验设置

### 自变量

- 传感器范围：`2 / 3 / 4 / 5`

### 控制变量

- 感知模式：`manhattan`
- 机器人数量：`3`
- 协作策略：`shared_map_reservation`
- 障碍密度：`0.15`
- 地图大小：`30 x 30`
- 目标覆盖率：`0.95`
- 动态障碍数量：`0`
- 噪声：`0.0 / 0.0`
- 每个条件重复运行：`10` 个随机种子

### 主要评价指标

- 最终覆盖率 `coverage_rate`
- 总步数 `total_steps`
- 重复访问次数 `duplicate_visit_count`
- 机器人间重叠格数 `inter_agent_overlap_cells`
- 达到不同覆盖阈值的步数
- 总能耗 `total_energy_used`
- 充电等待步数 `total_charge_wait_steps`

## 3. 主要结果

实验汇总结果如下：

- `range_2`
  - 平均总步数：`529.4`
  - 平均覆盖率：`0.9507`
  - 平均重复访问次数：`764.7`
  - 平均重叠格数：`256.4`
  - 平均总能耗：`1878.35`

- `range_3`
  - 平均总步数：`483.2`
  - 平均覆盖率：`0.9510`
  - 平均重复访问次数：`650.9`
  - 平均重叠格数：`223.2`
  - 平均总能耗：`1731.38`

- `range_4`
  - 平均总步数：`444.8`
  - 平均覆盖率：`0.9510`
  - 平均重复访问次数：`559.4`
  - 平均重叠格数：`193.2`
  - 平均总能耗：`1612.26`

- `range_5`
  - 平均总步数：`459.1`
  - 平均覆盖率：`0.9507`
  - 平均重复访问次数：`567.8`
  - 平均重叠格数：`210.0`
  - 平均总能耗：`1629.43`

覆盖推进速度方面：

- 达到 `80%` 覆盖的平均步数：
  - `range_2`: `399.5`
  - `range_3`: `364.8`
  - `range_4`: `329.2`
  - `range_5`: `338.0`

- 达到 `90%` 覆盖的平均步数：
  - `range_2`: `477.5`
  - `range_3`: `433.8`
  - `range_4`: `396.2`
  - `range_5`: `405.5`

- 达到 `95%` 覆盖的平均步数：
  - `range_2`: `528.4`
  - `range_3`: `482.2`
  - `range_4`: `443.8`
  - `range_5`: `458.1`

## 4. 结果分析

### 4.1 增大感知范围能够明显提升系统效率

从 `range_2` 到 `range_4`，系统总步数持续下降：

- `529.4 -> 483.2 -> 444.8`

这说明随着感知范围扩大，机器人能够更早获得周围环境信息，从而更高效地选择 frontier 和清扫目标，减少无效探索与回溯。

### 4.2 感知范围增加能够减少重复访问和重叠探索

重复访问次数和机器人间重叠格数也随着范围扩大而下降：

- 重复访问：`764.7 -> 650.9 -> 559.4`
- 重叠格数：`256.4 -> 223.2 -> 193.2`

这说明更大的感知范围有助于机器人更清楚地了解局部环境，减少多机器人在相似区域的重复行动。

### 4.3 `range_4` 是本组实验中的最佳点

在当前实验设置下，`range_4` 在多个指标上表现最好：

- 总步数最低
- 重复访问次数最低
- 重叠格数最低
- 总能耗最低

因此可以认为，在当前地图规模和障碍密度下，`sensor_range = 4` 是一个较优的感知范围设置。

### 4.4 感知范围并不是越大越好

从 `range_4` 增加到 `range_5` 后，系统并没有继续改善，反而在部分指标上略有回退：

- 总步数从 `444.8` 回升到 `459.1`
- 重复访问从 `559.4` 回升到 `567.8`
- 重叠格数从 `193.2` 回升到 `210.0`

这说明更大的感知范围虽然带来了更多信息，但并不一定继续转化为更高效率。  
在当前算法框架下，当感知范围达到一定程度后，额外信息的边际收益开始减弱。

### 4.5 能耗变化与效率变化基本一致

总能耗随感知范围从 `2` 增加到 `4` 持续下降：

- `1878.35 -> 1731.38 -> 1612.26`

而在 `range_5` 时略有回升到 `1629.43`。  
这与总步数和重复访问的变化趋势一致，说明更合理的感知范围不仅节省时间，也减少整体资源消耗。

## 5. 结论

本实验表明，传感器范围对多机器人清扫系统性能有显著影响。

主要结论如下：

1. 增大感知范围能够明显提升覆盖效率，减少达到目标覆盖率所需步数
2. 更大的感知范围能够减少重复访问和机器人间重叠探索
3. 在当前设置下，`sensor_range = 4` 表现最佳
4. 感知范围并不是越大越好，超过一定范围后性能提升会趋于饱和，甚至略有回退

## 6. 可直接用于报告的总结表述

可以在报告中写成如下风格：

> 实验结果表明，传感器范围对多机器人清扫系统的性能具有显著影响。在固定协作策略和环境参数下，随着感知范围从 2 增加到 4，系统在总步数、重复访问次数、机器人间重叠格数以及总能耗等指标上均表现出明显改善，说明更大的感知范围能够帮助机器人更高效地获取环境信息并减少无效探索。然而，当感知范围进一步扩大到 5 时，系统性能未继续提升，反而在部分指标上略有回退。这说明感知范围存在一个较优设置，在当前实验场景中，`sensor_range = 4` 是最合适的参数。
