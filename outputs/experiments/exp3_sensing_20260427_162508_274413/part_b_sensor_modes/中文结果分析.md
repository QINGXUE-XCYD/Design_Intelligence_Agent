# 实验3B结果分析：不同感知模式对系统性能的影响

## 1. 实验目的

本实验旨在比较不同感知模式对多机器人清扫系统性能的影响。  
在固定机器人数量、感知范围和协作策略的条件下，分析默认曼哈顿感知、欧氏距离感知以及带遮挡的可见域感知三种模式在覆盖效率、重复访问和能耗方面的差异。

## 2. 实验设置

### 自变量

- 感知模式：
  - `manhattan`
  - `euclidean`
  - `occluded_manhattan`

### 控制变量

- 传感器范围：`3`
- 机器人数量：`3`
- 协作策略：`shared_map_reservation`
- 障碍密度：`0.15`
- 地图大小：`30 x 30`
- 目标覆盖率：`0.95`
- 动态障碍数量：`0`
- 噪声：`0.0 / 0.0`
- 每个条件重复运行：`10` 个随机种子

### 主要评价指标

- 最终覆盖率 `coverage_rate`
- 总步数 `total_steps`
- 重复访问次数 `duplicate_visit_count`
- 机器人间重叠格数 `inter_agent_overlap_cells`
- 达到不同覆盖阈值的步数
- 总能耗 `total_energy_used`
- 充电等待步数 `total_charge_wait_steps`

## 3. 主要结果

实验汇总结果如下：

- `manhattan`
  - 平均总步数：`483.2`
  - 平均覆盖率：`0.9510`
  - 平均重复访问次数：`650.9`
  - 平均重叠格数：`223.2`
  - 平均总能耗：`1731.38`

- `euclidean`
  - 平均总步数：`478.8`
  - 平均覆盖率：`0.9505`
  - 平均重复访问次数：`631.4`
  - 平均重叠格数：`216.1`
  - 平均总能耗：`1707.90`

- `occluded_manhattan`
  - 平均总步数：`471.1`
  - 平均覆盖率：`0.9508`
  - 平均重复访问次数：`628.1`
  - 平均重叠格数：`211.9`
  - 平均总能耗：`1700.06`

覆盖推进速度方面：

- 达到 `80%` 覆盖的平均步数：
  - `manhattan`: `364.8`
  - `euclidean`: `366.3`
  - `occluded_manhattan`: `359.3`

- 达到 `90%` 覆盖的平均步数：
  - `manhattan`: `433.8`
  - `euclidean`: `431.1`
  - `occluded_manhattan`: `426.3`

- 达到 `95%` 覆盖的平均步数：
  - `manhattan`: `482.2`
  - `euclidean`: `477.8`
  - `occluded_manhattan`: `470.1`

## 4. 结果分析

### 4.1 三种感知模式都能稳定完成任务

三种模式的成功率均为 `1.0`，最终覆盖率都稳定在 `0.95` 左右。  
这说明在当前实验设置下，三种感知模式都具备完成任务的能力，系统不会因为更换感知方式而失去基本可用性。

### 4.2 `occluded_manhattan` 整体表现最好

从总步数、重复访问、重叠格数和总能耗四个指标综合来看，`occluded_manhattan` 表现最好：

- 总步数最低：`471.1`
- 重复访问次数最低：`628.1`
- 重叠格数最低：`211.9`
- 总能耗最低：`1700.06`

这说明带遮挡的可见域感知能够在不破坏系统稳定性的前提下，提高机器人对环境的感知质量，从而减少无效探索和重复劳动。

### 4.3 `euclidean` 模式略优于默认 `manhattan`

与默认 `manhattan` 相比，`euclidean` 模式在多个指标上有轻微改善：

- 总步数：`483.2 -> 478.8`
- 重复访问：`650.9 -> 631.4`
- 重叠格数：`223.2 -> 216.1`
- 能耗：`1731.38 -> 1707.90`

这说明仅仅改变感知范围的几何形状，也会对系统效率产生一定影响。

### 4.4 带遮挡感知的提升是稳定但不是极端的

虽然 `occluded_manhattan` 是本组最优，但它对默认模式的提升幅度并不是特别巨大，而是表现为一种稳定的小幅改善。  
这意味着：

- 原有 `manhattan` 感知模型已经能够支持系统完成任务
- 更真实的可见域建模不会导致系统退化
- 加入遮挡后，系统性能可以进一步优化，但这种提升更接近“稳步改进”，而不是“完全改变系统行为”

这种结果对于 coursework 是有利的，因为它既表明新模型有价值，又避免了过于夸张、不易解释的结论。

### 4.5 为什么“不能透视”反而更有利

本实验中一个很值得讨论的现象是：更接近现实的带遮挡感知模式不仅没有降低效率，反而取得了最好的综合表现。  
一个合理解释是，在当前地图环境下，**空间上看起来较近的格子，并不一定在路径上真正容易到达**。

例如：

- 某些 frontier 或可清扫区域虽然在几何距离上离机器人很近
- 但它们可能被障碍物隔开
- 机器人真正到达这些位置时，需要绕行较长路径

在默认的“可透视”感知模式下，机器人会较早看到这些实际上需要绕路的区域，因此可能更频繁地把它们当作候选目标。  
这会带来两个问题：

1. 机器人容易优先选择“看起来近、实际上绕路远”的目标
2. 多个机器人会更早暴露于大量远端候选点，增加目标切换和重复探索的可能性

而在带遮挡的感知模式下，机器人只能优先看到当前真正有视线可达的区域。  
因此系统会更倾向于：

- 逐步推进局部可见空间
- 先处理当前更直接、路径更合理的目标
- 等真正绕到障碍后方时，再发现后面的新区域

这使得目标选择更贴近真实可达性，而不是仅仅依赖表面上的几何接近程度。  
从结果上看，这种限制反而减少了无效规划和重复访问，因此整体效率更高。

### 4.6 感知模式影响主要体现在效率和重复访问，而非成功率

由于三种模式都能完成任务，因此本实验的差异主要体现在：

- 完成任务需要多少步
- 是否存在更多重复访问
- 多机器人之间是否更容易探索重叠区域
- 系统是否更省能耗

因此在报告中应强调：感知模式影响的是**系统效率和协作质量**，而不是“能不能完成任务”。

## 5. 结论

本实验表明，不同感知模式会对多机器人清扫系统的效率产生可观影响。

主要结论如下：

1. 三种感知模式都能稳定完成任务
2. `euclidean` 相比默认 `manhattan` 有小幅性能提升
3. `occluded_manhattan` 综合表现最好，在总步数、重复访问、重叠格数和能耗方面均优于其他模式
4. 带遮挡的可见域感知能够提高系统感知 realism，同时带来稳定但适度的性能改进
5. 一个可能原因是，遮挡限制减少了机器人对“几何距离近但真实需要绕路”的区域的过早偏好，使目标选择更贴近真实可达性

## 6. 可直接用于报告的总结表述

可以在报告中写成如下风格：

> 实验结果表明，不同感知模式会对多机器人清扫系统的性能产生一定影响。在固定感知范围和协作策略的条件下，默认的曼哈顿感知模型、欧氏距离感知模型以及带遮挡的可见域感知模型均能够稳定完成任务，但在效率指标上存在差异。其中，欧氏距离感知相较默认模型表现出轻微优势，而带遮挡的可见域感知在总步数、重复访问次数、机器人间重叠格数以及总能耗方面表现最佳。一个合理解释是，在当前障碍环境中，一些空间上看似较近的目标实际上需要绕路到达；可透视感知会使机器人过早关注这些区域，而带遮挡感知则更倾向于优先处理当前真正可见、路径更合理的目标，因此整体协作效率更高。
