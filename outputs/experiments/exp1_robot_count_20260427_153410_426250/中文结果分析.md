# 实验1结果分析：机器人数量对清扫效率的影响

## 1. 实验目的

本实验旨在研究在固定协作策略条件下，机器人数量变化对多机器人清扫系统整体效率的影响。  
实验关注的问题是：随着机器人数量从 1 增加到 4，系统是否能够更快完成环境覆盖任务，以及这种提升是否伴随着重复访问、机器人间重叠或资源竞争的变化。

## 2. 实验设置

### 自变量

- 机器人数量：`1 / 2 / 3 / 4`

### 控制变量

- 协作策略：`shared_map_reservation`
- 传感器范围：`3`
- 障碍密度：`0.15`
- 地图大小：`30 x 30`
- 目标覆盖率：`0.95`
- 动态障碍数量：`0`
- 充电站容量：`1`
- 每个条件重复运行：`10` 个随机种子

### 主要评价指标

- 最终覆盖率 `coverage_rate`
- 总步数 `total_steps`
- 总路径长度 `total_path_length`
- 重复访问次数 `duplicate_visit_count`
- 机器人间重叠格数 `inter_agent_overlap_cells`
- 总能耗 `total_energy_used`
- 充电等待步数 `total_charge_wait_steps`

## 3. 主要结果

实验汇总结果如下：

- `1` 个机器人：平均总步数 `1390.20`，平均覆盖率 `0.9189`，成功率 `0.50`
- `2` 个机器人：平均总步数 `700.10`，平均覆盖率 `0.9508`，成功率 `1.00`
- `3` 个机器人：平均总步数 `483.20`，平均覆盖率 `0.9510`，成功率 `1.00`
- `4` 个机器人：平均总步数 `353.70`，平均覆盖率 `0.9508`，成功率 `1.00`

进一步看覆盖进程指标：

- 达到 `80%` 覆盖的平均步数分别为：
  - `1` 个机器人：`1091.5`
  - `2` 个机器人：`513.9`
  - `3` 个机器人：`364.8`
  - `4` 个机器人：`274.2`

- 达到 `90%` 覆盖的平均步数分别为：
  - `1` 个机器人：`1214.57`
  - `2` 个机器人：`613.2`
  - `3` 个机器人：`433.8`
  - `4` 个机器人：`324.0`

- 达到 `95%` 覆盖的平均步数分别为：
  - `1` 个机器人：`1279.4`
  - `2` 个机器人：`699.1`
  - `3` 个机器人：`482.2`
  - `4` 个机器人：`352.7`

## 4. 结果分析

### 4.1 机器人数量增加能够显著提升清扫效率

从平均总步数可以看出，随着机器人数量增加，系统完成目标覆盖率所需的步数明显下降。

- 从 `1` 个机器人增加到 `2` 个机器人时，总步数几乎减半
- 从 `2` 个机器人增加到 `3` 个机器人时，效率继续显著提升
- 从 `3` 个机器人增加到 `4` 个机器人时，仍然有提升，但增益相较前两次有所减弱

这说明多机器人协作确实能够提高覆盖效率，原因在于多个机器人可以并行探索和清扫不同区域，从而减少单个机器人独立完成任务所需的时间。

### 4.2 单机器人在当前设置下稳定性不足

`1` 个机器人条件下平均覆盖率只有 `0.9189`，成功率仅为 `0.50`，明显低于多机器人条件。  
这说明在当前地图大小、障碍密度、传感器范围和电量机制下，单机器人往往难以稳定达到 `95%` 目标覆盖率。

相比之下，`2` 个及以上机器人条件的成功率均为 `1.00`，表明系统一旦具备基本的多机器人协作能力，其任务完成稳定性会显著提高。

### 4.3 覆盖速度随着机器人数量增加而持续提高

从 `80%`、`90%`、`95%` 三个覆盖阈值的到达时间可以看出，机器人数量越多，覆盖曲线爬升越快。  
尤其是在前期探索阶段和中后期补扫阶段，多机器人并行作业都能带来明显加速效果。

因此，在报告中可以结合 `coverage_curves_by_robot_count.png` 说明：

- 多机器人不仅提升最终完成效率
- 还能够在整个任务过程中持续提高覆盖推进速度

### 4.4 机器人数量增加并不一定带来更高总能耗

总能耗均值分别为：

- `1` 个机器人：`1701.69`
- `2` 个机器人：`1689.98`
- `3` 个机器人：`1731.38`
- `4` 个机器人：`1671.17`

可以看到，总能耗并没有随着机器人数量线性增长，反而整体保持在相近范围内。  
这说明虽然机器人数量增多会引入更多主体，但由于任务完成更快、路径更高效，系统总体资源消耗并没有明显恶化。

### 4.5 更多机器人会引入更多协作重叠和充电等待

在多机器人条件下，机器人间重叠格数和充电等待步数开始出现：

- `2` 个机器人：重叠 `180.7`，等待 `26.5`
- `3` 个机器人：重叠 `223.2`，等待 `44.7`
- `4` 个机器人：重叠 `231.1`，等待 `68.4`

这说明随着机器人数量增加，虽然总体效率提高，但系统内部的协作冲突和共享资源竞争也会增加。  
尤其是 `4` 个机器人条件下，虽然总步数最低，但充电等待步数明显高于 `2`、`3` 个机器人。

因此可以认为：

- 机器人数量增加带来显著效率提升
- 但也会逐渐增加资源竞争和协作开销

## 5. 结论

本实验表明，在固定协作策略 `shared_map_reservation` 下，增加机器人数量能够显著提升多机器人清扫系统的覆盖效率和任务完成稳定性。

主要结论如下：

1. `1` 个机器人在当前设置下难以稳定完成任务，成功率只有 `50%`
2. `2` 个及以上机器人均能稳定达到目标覆盖率
3. 机器人数量增加会显著减少完成任务所需步数，并加快覆盖率提升速度
4. 从 `3` 个机器人增加到 `4` 个机器人时仍有提升，但边际收益开始减弱
5. 更多机器人会带来更高的协作重叠和充电等待，因此系统性能存在“效率提升”和“资源竞争增加”之间的权衡

## 6. 可直接用于报告的总结表述

可以在报告中写成如下风格：

> 实验结果表明，在共享地图与目标预留协作策略下，增加机器人数量能够显著提高系统覆盖效率。与单机器人相比，多机器人系统不仅在平均总步数上有明显下降，而且在达到 80%、90% 和 95% 覆盖率的时间上均有稳定改善。其中，单机器人在当前环境设置下仅有 50% 的成功率，而 2 个及以上机器人均能稳定完成任务，说明多机器人协作显著提高了系统鲁棒性。与此同时，随着机器人数量进一步增加，机器人间重叠访问和充电等待步数也有所上升，表明系统在获得更高效率的同时，也面临更明显的资源竞争与协作开销。
