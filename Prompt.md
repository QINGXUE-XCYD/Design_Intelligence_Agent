二、项目目标
====================

请帮助我设计并逐步实现一个“多扫地机器人协同清扫系统”。系统运行在二维室内仿真环境中，环境可以是未知或部分已知的，包含静态障碍物和动态障碍物。多个扫地机器人作为独立 autonomous agents，需要通过感知、探索、建图、路径规划、局部避障和协同分工，尽可能高效地完成清扫任务。

核心目标包括：
1. 多机器人在未知或部分已知环境中的自主探索
2. 对环境进行建图或更新可用地图表示
3. 协同完成区域覆盖与清扫
4. 避免静态障碍物与动态障碍物
5. 通过不同 coordination / communication / task allocation 策略进行实验对比
6. 输出可用于 coursework report 和 presentation 的实验结果、图表和分析结论

====================
四、推荐实现思路
====================

请优先采用以下总体设计：

【环境表示】
- 使用二维栅格地图（occupancy grid）
- 地图状态可包含：
  - 未知区域
  - 空闲区域
  - 静态障碍
  - 动态障碍
  - 已清扫区域
  - 待清扫区域
  - 可选：充电站

【机器人 Agent】
每个机器人是一个独立 agent，至少应包含：
- 唯一 ID
- 当前位置
- 局部感知能力
- 局部地图 / 或共享地图访问能力
- 当前目标
- 决策逻辑
- 路径规划接口
- 局部避障逻辑
- 清扫动作
- 协同策略接口

【推荐的核心循环】
每个仿真 step 中，每个机器人执行：
1. 感知环境
2. 更新内部状态 / 地图
3. 与其他机器人通信（若策略需要）
4. 选择目标区域或任务
5. 路径规划
6. 局部避障 / 必要时重规划
7. 移动并执行清扫
8. 记录性能指标

【推荐算法】
- 探索：frontier-based exploration
- 路径规划：A* 作为默认方案
- 协同：
  - baseline 1：无通信
  - baseline 2：共享地图
  - baseline 3：共享地图 + goal reservation
  - advanced：区域分配或 auction-based task allocation
- 局部避障：
  - 简单反应式避障
  - 路径阻塞时重新规划

====================
六、预期实验问题
====================

请将系统设计成能够自然支持以下 3 类研究问题：

问题1：机器人数量变化是否能显著提升清扫效率？
- 对比 1/2/3/4 个机器人
- 指标：总完成时间、覆盖率、重复覆盖率、路径长度

问题2：通信或共享地图是否提升协作效果？
- 对比：
  - 无通信
  - 周期性地图共享
  - 全局共享地图
- 指标：覆盖率、冲突次数、重复率、完成时间

问题3：不同任务分配/协同策略的效果如何？
- 对比：
  - 无显式分配
  - goal reservation
  - 区域划分
  - 可扩展：auction-based allocation
- 指标：整体效率、负载均衡、冲突情况、鲁棒性
